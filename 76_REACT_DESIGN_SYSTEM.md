# 76_REACT_DESIGN_SYSTEM

## Purpose

Defines the React design system direction for Evercrafted Studio.

---

## Visual Doctrine

Evercrafted should feel:
- premium
- editorial
- intelligent
- botanical
- structured
- calm

Evercrafted OS can feel darker and more technical.
Evercrafted Studio should be creator-friendly but still premium.

## Components

Required MVP components:
- Button
- Card
- Panel
- Badge
- Input
- Textarea
- Select
- Tabs
- Modal
- JsonViewer
- StatusIndicator
- EmptyState
- LoadingState

---

## Interaction Standards

- Buttons should have clear active/disabled/loading states.
- Panels should be structured and readable.
- Advanced data should be collapsible.
- JSON views should never dominate normal creator workflows.
