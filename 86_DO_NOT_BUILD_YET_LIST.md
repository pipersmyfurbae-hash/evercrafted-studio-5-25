# 86_DO_NOT_BUILD_YET_LIST

## Purpose

This file prevents scope creep.

These systems are important, but they should not be built in MVP V1.

---

# Do Not Build in MVP

## Marketplace

Do not build:
- blueprint sales
- creator payouts
- licensing
- marketplace browsing
- public creator profiles

Reason:
The blueprint creation engine must work first.

---

## Moodoor

Do not build:
- consumer quiz
- consumer matching
- door preview
- consumer checkout

Reason:
Moodoor depends on a working Evercrafted OS and blueprint library.

---

## Adaptive Intelligence

Do not build:
- learning system
- user-specific adaptive model
- marketplace performance prediction
- formula evolution

Reason:
Need real usage data first.

---

## Blueprint Genome Advanced Features

Do not build:
- mutation system
- inheritance tree
- family tree UI
- branch comparison

Reason:
V1 only needs saved blueprint versions.

---

## Enterprise Systems

Do not build:
- team permissions
- B2B licensing
- hospitality workflows
- approval systems

Reason:
Founder-led MVP should validate creator workflow first.

---

## Full Admin OS

Do not build:
- full internal admin console
- moderation tools
- analytics dashboards
- marketplace admin

Reason:
Use uploaded console as architecture reference, not MVP requirement.

---

## Integrations

Do not build yet:
- Shopify publishing
- Etsy API
- Gmail
- Calendar
- Stripe Connect
- file storage pipelines

Reason:
These add complexity before core value is proven.

---

# Build First

Only build:

Memory → Blueprint → Placement → Visualizer → Save → Build Sheet → Render Prompt
