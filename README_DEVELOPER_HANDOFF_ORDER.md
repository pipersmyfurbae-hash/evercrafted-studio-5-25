# README_DEVELOPER_HANDOFF_ORDER.md

# What to Send Developers

There are three different kinds of files in the Evercrafted package.

## 1. Canon Files
These explain what Evercrafted is.

Send these first so developers understand the platform:
- 00_EVERCRAFTED_UNIFIED_ECOSYSTEM_CANON.md
- 04_EVERCRAFTED_SYSTEM_ARCHITECTURE.md
- 28_ENGINE_LAYER_MAP.md
- 29_PROCEDURAL_PLACEMENT_DOCTRINE.md

## 2. Engine Specs
These explain how the engine works.

Send these when developers begin backend/frontend architecture:
- 05_EC_CANON_V1_BLUEPRINT_SCHEMA.md
- 06_EMOTION_ENGINE_SPEC.md
- 07_FORMULA_ENGINE_SPEC.md
- 08_WREATH_DNA_ENGINE_SPEC.md
- 09_PLACEMENT_ENGINE_SPEC.md
- 10_PLACEMENT_VISUALIZATION_ENGINE_SPEC.md
- 11_BLUEPRINT_COMPILER_SPEC.md
- 12_BUILDBLOOM_MANUFACTURING_SPEC.md
- 13_INVENTORY_ENGINE_SPEC.md

## 3. MVP Build Files
These tell developers exactly what to build first.

These are the actual build instructions:
- 03_EVERCRAFTED_MVP_SCOPE.md
- 15_DESIGN_STUDIO_PRD.md
- 20_API_CONTRACTS.md
- 21_DATABASE_SCHEMA.md
- 22_MVP_SPRINT_PLAN.md
- 23_TECH_STACK_AND_DEPLOYMENT.md
- 24_BUSINESS_RULES_AND_VALIDATION.md

## Important

The files created in this pack are architecture/canon bridge files. They are necessary, but they are not the full build ticket by themselves.

The developer build package still needs:
- MVP Scope
- API Contracts
- Database Schema
- Sprint Plan
- Acceptance Criteria

Those are the files that tell developers exactly what to build.
