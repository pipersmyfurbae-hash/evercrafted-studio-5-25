# 38_SPRINT_1_FOUNDATION_TICKETS

## Sprint Goal

Create a stable production-ready MVP foundation.

This sprint is NOT about beautiful UI.

This sprint is about:
- structure
- repo organization
- app startup
- backend startup
- shared types
- database foundation
- environment setup

---

# Ticket 1.1 — Frontend Foundation

## Goal

Initialize frontend app.

## Requirements

- Next.js
- TypeScript
- Tailwind
- App Router
- Clean folder structure

## Acceptance Criteria

- Frontend runs locally.
- No broken starter code.
- Clean structure exists.

---

# Ticket 1.2 — Backend Foundation

## Goal

Initialize backend API.

## Requirements

- FastAPI
- Health route
- Environment loading
- API version prefix

## Acceptance Criteria

- Backend runs locally.
- `/api/v1/health` returns healthy response.

---

# Ticket 1.3 — Shared Types

## Goal

Create shared foundational types.

## Acceptance Criteria

- Shared type package exists.
- Blueprint base types exist.
- Emotion types exist.
- Placement types exist.

---

# Ticket 1.4 — Database Foundation

## Goal

Prepare database schema foundation.

## Acceptance Criteria

- SQL schema exists.
- User table exists.
- Blueprint table exists.
- Placement table exists.

---

# Ticket 1.5 — Repo Standards

## Goal

Prepare production repo standards.

## Acceptance Criteria

- `.gitignore` exists
- docs folder exists
- apps folder exists
- packages folder exists
- clean naming conventions used

---

# Important Rule

Do NOT build:
- Marketplace
- Moodoor
- evaluator systems
- adaptive intelligence
- advanced rendering
- fake dashboards
- mockup-only UI
